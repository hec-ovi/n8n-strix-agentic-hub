n8n Strix Agentic Hub
=====================

Purpose
- Self-hosted automation stack built around `n8n`.
- Local report generation service backed by `Ollama` and `gpt-oss:20b`.
- Output artifacts stored on the host and delivered through SMTP.

Tech Used
- `n8n` for orchestration.
- `FastAPI` for the backend service.
- `Ollama` with `gpt-oss:20b` for local inference.
- `PostgreSQL` and `Redis` for queue-mode n8n.
- `Mailpit` for local SMTP testing.
- `ReportLab` for PDF rendering.
- `Docker Compose` for local deployment.

Current Flows
- `Webhook -> n8n -> report-service -> Ollama -> Markdown/PDF -> email`
- `Telegram-style webhook -> n8n -> report-service -> Markdown/PDF -> email`

Design Intent
- Keep `n8n` focused on triggers, flow control, retries, and delivery.
- Keep AI calls, PDF generation, and SMTP work in the backend service.
- Keep the model layer behind an OpenAI-compatible chat interface so it can be swapped later with less friction.
- Keep workflows versioned as JSON artifacts under `n8n/workflows`.

Production Direction
- Scale workers first.
- Move binary artifacts to object storage when local disk stops being enough.
- Put proper HTTPS ingress in front of public webhooks.
- Treat Mailpit as local-only and replace it with real SMTP for external delivery.
